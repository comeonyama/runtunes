# RunTunes 要件定義書 v1.0

## 1. プロジェクト概要

### 1.1 システム名

RunTunes

### 1.2 目的

ランナーがランニングシーンに最適なSpotifyプレイリストを短時間で作成できるWebサービスを提供する。

### 1.3 対象ユーザー

-   ランナー
-   Spotify利用者
-   DiscoverRoutes.jp利用者

## 2. システム範囲

### 対象機能

-   ランニングシーン選択
-   年代選択
-   ジャンル選択
-   曲数選択
-   人気度設定
-   AIによる候補曲生成
-   候補曲の削除・並び替え
-   Spotifyプレイリスト作成

### 対象外

-   音楽再生
-   SNS機能
-   独自音楽配信

## 3. 機能要件

### FR-01 条件指定

シーン・年代・ジャンル・曲数・人気度を指定できる。

### FR-02 AI選曲

OpenAI APIを利用して候補曲を生成する。

### FR-03 候補曲編集

候補曲の削除・並び替えを行える。

### FR-04 Spotify保存

Spotifyアカウントへプレイリストを保存できる。

### FR-05 キャッシュ

AI生成結果およびSpotify検索結果をDBへキャッシュする。

## 4. 非機能要件

-   初回表示3秒以内
-   HTTPS通信
-   OAuth認証
-   APIキーはサーバー側管理
-   レスポンシブ対応
-   保守性を考慮したReact + TypeScript構成

## 5. 制約

-   Spotify Web API利用
-   OpenAI API利用
-   WordPress固定ページ配下でSPA提供
-   MySQLをWordPressと共用

## 6. 成功指標

-   30秒以内にプレイリスト作成
-   Spotify API呼び出しをキャッシュで削減
-   MVP公開
